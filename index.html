作用域、闭包与变量提升

发表于 2016-07-10   |   分类于 JavaScript   |  

作用域

作用域是指当前代码可以访问到的变量的一个环境，按照嵌套关系来讲，在外层的作用域中不能访问内部作用域中的变量，但内部作用域可以访问到外部作用域中的变量。

全局作用域
window对象是javascript中最外层的一个作用域，在任何地方都可以访问全局作用域中的变量。

函数作用域
每一个函数在创建时(不是在调用时)可以产生一个内部作用域，在函数内部访问指定的变量时，如果函数内部不存在该变量，则会一层一层向上级作用域查找，一直到全局作用域。

实例

闭包

闭包就时在函数中返回一个内部函数，在执行这个内部函数时可以通过作用域访问到函数中的变量。

保存局部变量
函数中的局部变量在运行完后就会被销毁，可以使用闭包方法来保存函数内部的变量。

实例

变量提升

变量提升是指在使用var关键字声明变量并赋值的过程会被解析为先使用var关键字在当前作用域的最前面声明这个变量，再等代码执行到赋值的语句时再进行赋值。

参数二

在JavaScript中发生数据类型转换的场景分为人为主动进行类型转换与在一些运算符与语句中隐式类型转换。

主动类型转换

转换为数值

可以使用Number函数将任何类型的数据转换成数值。

转换为字符串

Boolean方法可以将任何类型数据转换为布尔值。

默认转换为字符串

JavaScript类型判断

发表于 2016-07-06   |   分类于 JavaScript   |  

目前知道的JavaScript中类型判断有以下四种方法，其中使用constructor属性这种方法稍微稍微加工下就可以判断出所有的数据类型，而其他的方法都不能做到。

typeof

typeof操作符在一般情况下可以判断出undefined、boolean、number、string、function这几种数据类型，其他的类型都会被认定为object类型。

但是在使用对象创建法创建基本类型数据的情况下，typeof操作符只能判断出undefined与function这两种数据类型，其他的都会被认定为object类型。

instanceof

instanceof操作符用来判断指定对象原型链上的constructor属性是否为指定的函数(基本上很少使用)，如果是则返回true，否则返回false。只能用来判断JavaScript的内置对象与自定义对象，用来判断基本数据类型则全部返回false(如果使用对象创建法创建的基本类型数据则可以来判断)。

Object.prototype.toString

Object.prototype.toString这个方法可以将对象转为其字符串表示法，可以用来判断基本数据类型与内置对象，但是自定义对象会被判断为object。

constructor

constructor属性指向的是这个对象的构造函数，除了undefined与null这两种数据类型不能使用以外，其他的所有数据类型都可以使用。因为在JavaScript中，除了undefined与null以外，其他所有的数据类型都是对象，而对象都是有其构造函数的，constructor这个属性就指向了该对象的构造函数。
